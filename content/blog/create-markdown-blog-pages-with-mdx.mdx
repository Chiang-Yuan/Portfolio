Creating blog pages with various components could be an awful experience in web development. With the state-of-the-art MDX in Gatsby.js framework, importing plugins and components is no longer a pain in the butt for a self-taught programmer like me. All of the good web design elements, such as stylish typography, YouTube embedded video, Latex rendered math equation, etc., could be easily done in simple and straightforward fashions.

The basic markdown syntax can be found on [official website]([Markdown](https://www.markdownguide.org/basic-syntax/)
). Below I show the basic components as well as some extensions for the scientific and technical blog pages with full functions and features:

## List

To create bullet and numbered items in list format, simply use `*`, `+`, `-`, `1.` along with tabs:

```
* Gatsby
  * docs
- Gatsby
  - docs
+ Gatsby
  + docs

1. One
1. Two
1. Three
```
The above code will be rendered as:

* Gatsby
  * docs
- Gatsby
  - docs
+ Gatsby
  + docs

1. One
1. Two
1. Three

## Video

To embed videos in MDX (like YouTube, vimeo), we need to install `gatsby-remark-responsive-iframe` through terminal
```
npm install --save gatsby-remark-responsive-iframe
```
Reponsive iframe has one biggest advantage: scaling the frame with fixed aspect ratio with respect to screen size. After installation, add the following lines in `gatsby-config.js`
```
plugins: [
  {
    resolve: `gatsby-plugin-mdx`,
    options: {
      gatsbyRemarkPlugins: [
        `gatsby-remark-responsive-iframe`,
      ],
    },
  }
]
```
In the MDX file, add ***embedding link*** of YouTube video to `src` and assign attributes `width` and `height` for aspect ratio calculation.
```
<iframe src="https://www.youtube.com/embed/69yc9KvO2VU" width="880" height="495"></iframe>
```

## Use MathJax to render Latex

[MathJax](https://www.mathjax.org/) provides painless accessibility of beautiful math rendering in all browsers.

$$
\begin{equation}
f(x) = \int_{-\infty}^\infty \hat f(\xi) e^{2 \pi i \xi x} d\xi
\end{equation}
$$

Importing MathJax into Gatsby.js needs the plugin `gatsby-remark-mathjax-ssr`

```
npm install --save gatsby-remark-mathjax-ssr
```
Same as the responsive frame for video embeddings, we need to specify the plugin in `gatsby-config.js`
```
plugins: [
  {
    resolve: `gatsby-plugin-mdx`,
    options: {
      gatsbyRemarkPlugins: [
        `gatsby-remark-mathjax-ssr`,
      ],
    },
  }
]
```
